## The Archivists' Practice

This page complements the [Archive Model](/archive-model/). It shows how Archivists turn recordings into structured, reusable archive items in practice.

---

### What the Archivist Does

The Archivist works with recordings from events, workshops, and community sessions.

They:
* organize and structure files
* assign stable identifiers
* create consistent metadata
* ensure recordings are easy to find and reuse over time

The role combines careful execution with independent judgment. Archivists regularly work with incomplete inputs and bring clarity and structure to them.

---

### How Work Happens

Work is primarily async.

Tasks come from:
* a shared task list
* coordination with organizers and contributors

Archivists also:
* identify gaps
* improve structure and consistency
* document better approaches as they emerge

---

### How the Archive is Structured

The archive has two layers:

* **Storage** — raw recordings and files 

* **Metadata** — structured information about those recordings

Files are stored in shared storage, organized by event or project.

Metadata is stored as YAML in a repository. Each YAML file references the corresponding files in storage, linking structure to content.

The system is simple, portable, and evolves over time.

---

### What a Completed Archive Item Includes

Each recording becomes an archive item with:

* organized, consistently named files
* a stable identifier
* a YAML metadata file with:
  * subject
  * speakers
  * event
  * date
  * description
  * topics
  * relevant links 
* an entry in the archive inventory

A completed item is retrievable, understandable, and ready to use.

Additional layers (summaries, timestamps, thematic grouping) can be added later.

---

### Workflow

For each recording, the Archivist:

1. assigns a stable ID
2. organizes and renames files
3. creates a YAML metadata file
4. applies consistent naming and vocabulary
5. ensures the item is complete and usable

This creates a reliable foundation for the archive.

---

### Working Alongside the Storyteller

The Archivist and Storyteller operate in parallel:

* Archivist → structure and consistency
* Storyteller → narrative and discoverability

Both work on the same archive. The Archivist ensures each recording is structured so it can be explored, interpreted, and connected over time.

---

### Tools and Approach

The system uses simple tools:

* shared storage for files
* a task list for coordination
* a YAML repository for metadata

Naming conventions are applied consistently.

The goal is a system that is:

* easy to understand
* easy to maintain
* able to scale

---

### Principles

* Keep it simple and structured
* Prioritize consistency over complexity
* Archive everything before enhancing it
* Improve the system while using it
* Favor reversible decisions

---

### Why This Matters

Without structure, recordings are hard to use.

With structure, they become part of a shared, growing body of knowledge.

The Archivist makes that possible.
